## 1. 이 문서를 읽는 사람에게

**한 줄 요약:** 베리시 교환 건을 한눈에 보고, 출고가 지연된 고객에게 **알림톡(카카오 메시지)을 버튼 한 번으로 보내는** 사내용 웹 대시보드입니다.

**목적:** 클로드(Claude) 계정이 바뀌어도 같은 맥(Mac)에서 이 폴더를 열면 **추가 설치 없이 바로 이어서** 쓸 수 있도록 안내합니다. 파일·키·설정·스킬은 디스크에 그대로 남아 있으니, 아래 순서만 따라 하면 됩니다.

**가장 중요한 한 가지:** 이 대시보드는 **자동으로 켜지지 않습니다.** 쓰려면 매번 직접 서버 2개를 켜야 합니다 (2번 항목). cron(자동 실행)에 등록돼 있지 않은 것을 확인했습니다.

`터미널(Terminal)` 앱을 열고, 아래를 **각각 다른 탭**에 붙여넣고 엔터:

**탭 1 — 백엔드(데이터 연결 서버)**
```bash

```
→ `Server running at http://localhost:3001` 이 보이면 성공.

**탭 2 — 화면(프론트엔드)** (탭 1은 켜둔 채로 `Command + T`로 새 탭)

→ `Local: http://localhost:5173/` 이 보이면 성공.

### 접속
크롬(Chrome) 주소창에 **http://localhost:5173/** 입력.

각 탭에서 `Control + C`. 또는 한 번에:
```bash
lsof -ti:3001 | xargs kill ; lsof -ti:5173 | xargs kill
```

> ⚠️ 두 터미널 탭은 **창을 닫으면 같이 꺼집니다.** 켜둔 채로 두세요. (클로드에게 켜달라고 하면 작업 세션이 끝날 때 같이 종료될 수 있어, 직접 켜는 쪽이 안정적입니다.)

## 3. 이게 무슨 도구인가 (처리 흐름)

### 데이터의 원천
모든 데이터는 **하나의 구글시트**에서 옵니다 (9번 참고). 대시보드는 이 시트를 **읽고**, 일부 셀(출고일·지불방법·교환옵션)은 화면에서 **고쳐 쓸 수도** 있습니다.

### 알림톡 발송 흐름 (핵심)
"교환 미처리 관리" 화면에서 **📨 발송** 버튼을 누르면:

```
브라우저 화면 (5173)
   │  발송 버튼 클릭
   ▼
백엔드 서버 (3001, cx-exchange-dashboard/server/index.js)
   │  POST /api/send-alimtalk
   ▼
파이썬 실행 (/usr/bin/python3 send_single_python.py)
   │  주문 상태(입고/미입고 × 옵션누락 × 미입금)에 따라 분기
   ▼
alimtalk_core.py  (메시지 템플릿 + 발송 함수)
   │
   ▼
루나소프트(Lunasoft) 알림톡 API → 고객 휴대폰
```

- **입고 여부 × 옵션 누락 여부 × 미입금 여부** 조합으로 8가지 메시지 템플릿(50195~50201) 중 하나가 자동 선택됩니다. 분기 로직은 `send_single_python.py` 39~70줄.
- 발송 결과는 `cx-exchange-dashboard/server/alimtalk.log` 에 쌓입니다.
- 발송 이력(언제 보냈는지)은 **브라우저 localStorage**에 저장됩니다. → **다른 브라우저/PC에서는 안 보입니다.** (서버에 저장되는 게 아님)

### 화면 구성 (사이드바 메뉴 = src/pages/)
개요, 자사몰 교환, 외부몰 교환, 불량 분석, 리드타임, 교환 실적, 재고 부족 알림, **교환 미처리 관리(StuckCases — 알림톡 발송이 여기 있음)**, 상품별 상세, 리포트 센터.

### 정상
- 서버 탭: `Server running at http://localhost:3001`
- 프론트 탭: `Local: http://localhost:5173/`
- 화면: 표에 교환 건들이 보이고, 새로고침(🔄) 버튼이 돈 뒤 데이터가 채워짐.

### 사람이 수동으로 확인해야 하는 케이스

## 7. 실행 옵션

- **기본:** 위 2번 방식(터미널 2탭). 가장 안정적.
- **클로드에게 부탁:** "백 프론트 실행해줘" 라고 하면 클로드가 켜줍니다. 단, 클로드 작업 세션이 끝나면 같이 꺼질 수 있어 **장시간 운영엔 부적합**.
- **포트:** 백엔드 `3001` 고정(`server/index.js:10`), 프론트 `5173`(Vite 기본). 다른 프로그램이 점유 중이면 충돌. 확인: `lsof -ti:3001`, `lsof -ti:5173`.
- **알림톡 단건 직접 테스트(개발용):** `send_single_python.py` 에 JSON 인자를 줘서 실행 가능하나, **실제 고객에게 발송되므로** 테스트는 본인 번호로만.
